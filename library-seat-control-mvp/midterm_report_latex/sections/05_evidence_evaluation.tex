The image algorithm module has the following evidence:

\begin{longtable}{|p{0.24\linewidth}|p{0.36\linewidth}|p{0.30\linewidth}|}
\hline

\hline

\hline

\hline

The local output file \texttt{latest\_seat\_status.csv} uses fields aligned with the data structure module: \texttt{seat\_id}, \texttt{floor}, \texttt{zone}, \texttt{detected\_at}, \texttt{has\_person}, \texttt{has\_object}, \texttt{status}, and \texttt{suspect\_duration}. A debug JSON file can also be generated to inspect raw YOLO detections.

One important finding from the school-library sample is that the general pre-trained YOLO model can produce false labels in library scenes. For example, a backpack or chair region may be confused with a person-like shape, and a desk/chair area may receive an irrelevant label. The current ROI logic reduces the impact of these mistakes on the final seat status, but real accuracy cannot be formally measured until real library video frames and ground-truth labels become available.
