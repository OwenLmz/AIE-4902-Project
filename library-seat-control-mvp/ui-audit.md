# 图书馆智能位控系统前端 UI 审计

审计日期：2026-06-25  
审计范围：当前 Streamlit 前端 `src/dashboard/app.py`，项目文档 `docs/PRD.md`、`docs/TECH.md`、`模块对接说明.md`，本地输出数据 `output/*.json`，示例配置 `src/data/*.json|csv`，数据库草案 `library_schema_fixed.sql`。

## 0. 审计执行情况

- 当前项目不是 Git 仓库，本阶段仅新增本审计文档，未修改前端代码。
- 已确认当前前端服务可访问：`curl -I http://localhost:8501` 返回 `HTTP/1.1 200 OK`。
- 浏览器自动化工具受运行环境元数据限制未能连接，因此页面组件以 Streamlit 源码、运行服务健康检查和输出 JSON 为准逐项审计。
- 当前前端没有发现 `Math.random`、`random`、`randint`、`uniform` 等随机业务数据生成。
- 当前前端只有一个页面：`图书馆智能位控 Dashboard`，未拆分学生端和管理员端。

## 2. 页面级问题

| 问题项 | 当前情况 | 影响 | 结论 |
|---|---|---|---|
| 学生端和管理员端信息混杂 | 同一页同时展示可用座位、人流拥挤、疑似占座数量、无人有物累计 | 学生可能看到偏管理员判断信息；管理员缺少可操作异常列表 | 修改 |
| 数据状态未标注 | 页面未显示“实时数据 / 模拟数据 / 暂无数据” | 当前 `output` 多来自 mock pipeline，用户可能误认为真实生产数据 | 修改 |
| API 来源缺失 | 前端直接读本地 JSON，没有调用 `/api/floor/{floor_id}/seats`、`/api/crowding` 等接口 | 与对接文档定义不一致，后续替换真实后端时需要重构数据层 | 待确认 |
| 空数据和错误状态不完整 | 缺少座位数据时有提示；人流数据缺失时默认显示 0、1、low | 错误或缺数据时可能展示伪造含义的数字 | 修改 |
| 座位布局表达不充分 | 使用 4 列卡片网格，不是真正基于 polygon/ROI 的座位布局图 | 学生无法理解座位空间位置，管理员也无法定位异常座位 | 修改 |
| 趋势图信息不完整 | 只有历史馆内人数折线，无标题单位、图例、时间范围、预测分界 | 不满足人流趋势决策要求 | 修改 |
| 状态枚举不完整 | 仅支持 `FREE`、`OCCUPIED`、`POSSIBLY_OCCUPIED`、`SUSPICIOUS`，未处理 `UNAVAILABLE` | 与后续不可用座位需求不完整 | 修改 |

## 6. 公式与字段一致性检查

| 指标 | 当前公式/来源 | 项目要求/文档 | 一致性 | 结论 |
|---|---|---|---|---|
| 座位状态 | 状态机输出：有人 `OCCUPIED`；无人有物累计达到阈值 `SUSPICIOUS`；无人无物 `FREE` | PRD 和 TECH 中一致 | 基本一致 | 保留 |
| 疑似占座持续时间 | `object_unattended_minutes` / `suspect_duration` | 文档中也有 `suspect_min` / `suspect_duration` | 字段名不统一 | 修改 |
| 当前馆内人数 | `sum(enter_count - exit_count)` 滚动计算 | SQL `gate_log.total_in_library`；API 文档 `current_num` | 语义一致，字段名不同 | 修改 |
| 拥挤等级 | `<40% low`、`<75% medium`、其余 high | 对接文档：`<50% low`、`50%-75% medium`、`75%-90% high`、`>90% full` | 不一致 | 待确认 |
| 座位占用率 | 当前前端未展示 | 要求：`occupied_seats / (total_seats - unavailable_seats)` | 缺失 | 待确认 |
| 未来 30 分钟趋势 | 当前不存在 | 学生端要求展示未来 30 分钟人数或趋势 | 缺失 | 待确认 |

## 7. 按用户角色的审计结论

### 学生端

当前学生能看到：

- 当前空闲座位数量；
- 当前使用中座位数量；
- 当前馆内人数；
- 图书馆容量；
- 拥挤等级；
- 每个座位的当前状态。

当前学生看不到或不够清楚：

- 数据是实时还是模拟；
- 哪个区域更容易找到座位，当前只有单个 A 区数据且没有区域概览；
- 未来 30 分钟人流趋势；
- 具体座位空间位置图；
- 点击座位后的详情；
- 人流图的单位、图例、时间范围。

### 管理员端

当前管理员能看到：

- 疑似占座总数；
- 每个座位的人/物检测结果；
- 无人有物累计分钟数。

当前管理员看不到或不能操作：

- 独立的疑似占座列表；
- 待处理状态；
- 优先级；
- 检测到的物品汇总；
- 确认占座、标记误判、暂不处理操作；
- 操作日志或处理结果；
- 点击列表后定位座位。
